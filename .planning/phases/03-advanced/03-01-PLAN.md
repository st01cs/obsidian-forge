## Key Patterns

**SUBG-01:** Sub-agents use `createAgentSession()` with `SessionManager.inMemory()` for isolation. Each sub-agent has its own session that doesn't share state with the main session.

**SUBG-02:** Agent definitions are `.md` files in `forge/agents/` with YAML frontmatter: `name`, `description`, `tools` (whitelist array), `model`, `maxTurns`. Body is the system prompt.

**SUBG-03:** Long-running sub-agents use `requestIdleCallback` to yield to the browser main thread, avoiding UI blocking.

## Agent Definition File Format

Each agent in `forge/agents/` follows this structure:
```text
name: <agent-name>
description: <description>
tools: [<tool1>, <tool2>, ...]
model: <model-name-or-default>
maxTurns: <number>
---
<system-prompt-content>
```

The frontmatter is parsed by `loadAgentDefinition()` in agent-loader.ts.
</context>
